### `/input/` - Input Images
Contains input images for the cryptographic schemes.

| File | Purpose | Size | Auto-Generated |
|------|---------|------|----------------|
| `input.pgm` | Secret image to encrypt | 200√ó200 | Yes (if missing) |
| `cover.pgm` | Cover image for DHCOD | 200√ó200 | Yes (if missing) |

**Format:** PGM (Portable Gray Map) - P2 (ASCII) or P5 (Binary)

**How to use custom images:**
1. Replace `input.pgm` with your secret image
2. Replace `cover.pgm` with your cover image
3. Ensure they're in PGM format
4. Run the program

## üîÑ Build and Run Workflow

### Step 1: Build
```bash
# Windows
build.bat

# Linux/macOS  
./build.sh

# Or use Makefile
make
```

**What happens:**
1. Compiler reads source files from `src/`
2. Creates executables in `bin/`
3. No changes to other directories

### Step 2: Run Main Program
```bash
# Windows
bin\vc_program.exe

# Linux/macOS
bin/vc_program
```

**What happens:**
1. Looks for `input/input.pgm` and `input/cover.pgm`
2. Creates them if missing
3. Generates shares in `output/vcs/`, `output/rg/`, `output/dhcod/`
4. Creates `output/binary_input.pgm`

### Step 3: Analyze Results
```bash
# Windows
bin\analyze.exe

# Linux/macOS
bin/analyze
```

**What happens:**
1. Reads images from `input/` and `output/`
2. Calculates metrics (entropy, file sizes, contrast)
3. Prints comparison table

## üìù File Naming Conventions

### Source Files
- Lowercase names: `main.cpp`, `vcs.cpp`
- Descriptive: `image_utils.cpp` (not `util.cpp`)
- Matching headers: `vcs.cpp` + `vcs.hpp`

### Documentation
- UPPERCASE for important docs: `README.md`
- Descriptive: `TECHNICAL_GUIDE.md` (not `TECH.md`)
- Consistent extension: All `.md` except validation report (`.txt`)

### Output Files
- Scheme prefix: `vcs_share1.pgm`, `rg_share1.pgm`
- Descriptive: `dhcod_share1_meaningful.pgm`
- Sequential numbering: `share1`, `share2`
